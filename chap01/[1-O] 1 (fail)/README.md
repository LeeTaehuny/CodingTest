### 1 [수학, 브루트포스 알고리즘, 정수론] (오답 : 시간초과, 입력 수 제한..)
---
https://www.acmicpc.net/problem/4375

[문제 풀이] 

- 시간초과 이유
- 1로만 이루어진 정수 num이 입력값 n으로 나눠 떨어질 때까지 num의 자리수를 증가시켰습니다.
- 즉, (num % n != 0) 이라면 (num = num * 10 + 1)을 반복하였고 cnt를 1씩 증가시켜 결국 나눠 떨어질 때 cnt를 출력하는 방식으로 구현했습니다.

- long long의 범위 또한 벗어나는 수치가 등장해 무한으로 반복하게 되는 로직이 되어 시간 초과가 결과로 나왔습니다.

[추가 사항]

- 이전 문제와 같은 모듈러 연산 규칙을 통해 해결 가능합니다.

- (a + b) % c = a % c + b % c
- (a * b) % c = a % c * b % c

- 즉, (num = num * 10 + 1) 이후 (num = num % n)을 통해 숫자를 줄여 계산하는 것으로 오버플로우를 방지할 수 있습니다.


- 또한 입력의 개수가 정해지지 않고 지속적으로 받아오기 때문에 EOF를 통해 입력을 받아줘야 합니다.

- 즉, while (scanf("%d", &n) != EOF) { } 구문을 통해 입력을 지속적으로 받는다는 것을 추가해줘야 합니다.
